Привет, Кот, вот мои ответы на вопросы

  1. Что представляют собой формы?
    - Набор из элементов управления, размещаемых на Web-странице;
    - Аналогичны диалоговым окнам Windows;
    - Обычные web-страницы.

    Ответ: первое - набор из элементов управления, размещаемых на Web-странице

  2. Для чего служат формы?

    Ответ: это основной инструмент для получения информации от пользователя и передачи ее на сервер.

  3. Каким типом задаётся поле ввода в форме текста?

    Ответ: type="text". Это может быть input или textarea с таким типом.

  4. Какими способами можно обратиться к элементу формы?

    Ответ:

    • по индексу элемента;
    • по имени элемента (атрибут name).

  5. Какой JS-код можно указать в атрибуте action?

    Ответ: код создания ссылки на нужную страницу или вызова функции отправки данных из формы. В сочетании с методом POST.

  6. Какой тип имеет свойство elements объекта Form?

    Ответ: массив.

  7. Как сделать валидацию номера кредитной карты?

    Ответ: js-код в файле.

    Часто для проверки валидности используют регулярные выражения. Но здесь обязательно нужно будет написать для пользователя,
    в каком формате надо вводить данные. И использовать регулярные выражения в зависимости от выбранного вида карты.

    Пример кода для ввода кредитной карты Visa:

    Введите номер кредитной карты.[Начиная с 4, длиной 13 или 16 цифр (Visa) and нажмите кнопку "Отправить"

  8. Восстановить значения умолчания в полях формы можно только по кнопке reset?

    Ответ: нет. Можно воспользоваться методом reset(), который можно применить к любой кнопке отправки формы или к форме в целом.

  9. Каким способом можно отправить данные из формы на сервер?

    Ответ: по кнопке Submit, методом submit(), по событию onSubmit.

  10. Как можно полностью перехватить и обработать javascript-функцией событие submit (отправку данных на сервер)?

    Ответ: используя метод evt.preventDefault(), который позволит отменить автоматическую перезагрузку страницы.

  11. Какие есть недостатки у стандартного способа задания валидации через HTML5?

    Ответ:

    • Применяются стандартные стили, которые нельзя изменить.
    • Эти стили применяются еще до того, как пользователь начнет работать с формой.
      То есть, например, обязательное для заполнения поле уже подсвечено красным.
    • Это довольно ограниченный способ проверки валидности. С его помощью можно проверить только факт заполнения полей,
      но не корректность введенных данных.
